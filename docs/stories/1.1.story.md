## User Story

**Como** um usuário que precisa realizar cálculos simples,
**Quero** uma calculadora web com as 4 operações básicas, suporte a teclado e display da expressão completa,
**Para que** eu possa efetuar cálculos diretamente no browser, sem instalar nada, com ou sem mouse.

---

## Acceptance Criteria

- [ ] AC1: Operações básicas — `2 + 3 = 5`, `10 - 4 = 6`, `6 × 7 = 42`, `15 ÷ 3 = 5` retornam resultado correto
- [ ] AC2: Precedência — `2 + 3 * 4 = 14` (multiplicação antes de adição)
- [ ] AC3: Divisão por zero — `5 ÷ 0` exibe "Erro: div/0" sem travar a aplicação
- [ ] AC4: Display duplo — expressão atual visível no topo; resultado em destaque abaixo
- [ ] AC5: Após `=`, display superior exibe `expressão = resultado` (ex: `12 × 3 = 36`)
- [ ] AC6: Teclado físico — dígitos 0–9, operadores `+ - * /`, `Enter` (=), `Backspace` (⌫), `Escape` (C)
- [ ] AC7: Botão C limpa tudo; botão ⌫ apaga último caractere da expressão
- [ ] AC8: Todos os botões têm `aria-label` descritivo; resultado anunciado via `aria-live="polite"`
- [ ] AC9: Navegação por Tab funciona em todos os botões em ordem lógica
- [ ] AC10: Testes unitários passam — `npm test` retorna exit code 0

## Dev Notes

### Arquitetura (ver `docs/architecture/architecture.md`)
- **Separação obrigatória:** funções `calculate()`, `parseTokens()` e `applyPrecedence()` são **puras** (sem acesso ao DOM) — exportadas para teste
- **Nunca usar `eval()`** — risco de segurança; usar parser de tokens próprio
- **ES modules:** `package.json` com `"type": "module"`; `calculator.test.js` usa `import { calculate } from './calculator.js'`

### Parser de tokens sugerido
```javascript
// Tokenize: '2 + 3 * 4' → ['2', '+', '3', '*', '4']
// applyPrecedence: processa * e / primeiro, depois + e -
// Retorna: number | CalcError
```

### Mapeamento de teclas no DOM
```javascript
const KEY_MAP = {
  'Enter': '=', 'Backspace': '⌫', 'Escape': 'C',
  '+': '+', '-': '-', '*': '*', '/': '/',
  '.': '.'
}
```
